Секционный контент

Секция article

Используется для самостоятельной части документа, которая имеет оконченный вид и может быть использована на других ресурсах. Это может быть статья или любой другой материал. Допустимо вкладывать один элемент article в другой article, если вложенный элемент логически связан с внешним.
Секция aside - используется для содержимого, косвенно связанного с основным. Например для комментариев, меток, сносок, боковых панелей и т.п.
Секция nav - используется для размещения навигации по содержимому.

Секция section

Используется для выделения некой логической области.

Фразовый контент

Введение во фразовый контент

К фразовому контенту относятся текстовые элементы разметки этого текста внутри абзаца, а также элементы которые могут быть помещены в текстовую фразу без разрыва на два абзаца.

Абзац можно создать директивно с помощью парного тега "P". При этом создаётся P-элемент, а текстовый узел абзаца становится дочерним для этого элемента. Сам P-элемент является потоковым, но не фразовым, и имеет блочное поведение. P-элементы нельзя вкладывать друг в друга - они могут быть только сестринскими.

Закрывающий тег /P не является обязательным. Браузер автоматически добавит его там, где абзац больше не может продолжаться (например перед очередным открывающим тегом p).

В отличие от элементов с блочным поведением, фразовые элементы имеют ограниченную ширину, определяемую шириной их содержимого. Каждый последующий фразовый элемент браузер размещает справа от предыдущего, т.о. заполняя всю ширину текущего блока либо экрана.

При отсутвии P-тегов браузер всё равно помещает текстовые фразы в отдельные узлы DOM, обрабатывая их как абзацы. Так например этот текст помещён в автоматически созданный абзац, не являющийся потомком для какого-то P-элемента.

Теги для расстановки переносов строки в тексте

При заполнении фразовым контентом всей ширины блока или страницы браузер игнорирует символы переноса строки, заменяя их пробелами. Так, например, это предложение в html-коде находится сразу после символа переноса строки, но при этом остаётся слитным с предыдущим предложением.


Текст после тега "BR" отображается с новой строки - потому что появился новый br-элемент, который разделил фразу на две фразы.

Вывод на экран служебных символов

Для вывода на веб-странице особых символов, которыми задаются теги и прочие элементы разметки, используются следующие специальные символьные последовательности:
< - Открывающая треугольная скобка.
> - Закрывающая треугольная скобка.
& - Знак амперсанда.
" - Двойные кавычки.
  - Неразрывный пробел.
­ - Мягкий дефис (отображаемый только в конце строки)
★ - UTF8-символ, заданный числом в десятичной системе исчисления.
₽ - UTF8-символ, заданный числом в шестнадцатиричной системе исчисления.

Гиперссылки

Во фразах можно указывать ссылки на другие страницы и сайты. Все потомки A-элемента становятся кликабельными и нажав на них можно перейти по ссылке.

Для любого элемента можно указать аттрибут id с уникальным значением. Тогда можно использовать якорную ссылку на этот элемент.

Аттрибутом target можно указать, в каком окне или фрейме будет открыта ссылка: Открыть ссылку в новом окне

Добавление атрибута download позволяет скачать файл по ссылке: Скачать фото
